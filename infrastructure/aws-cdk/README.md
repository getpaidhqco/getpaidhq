# Payloop AWS Infrastructure

This project contains the AWS CDK code to provision the infrastructure required for Payloop, specifically the Kinesis streams for event processing.

## Prerequisites

- Node.js 14.x or later
- AWS CLI configured with appropriate credentials
- AWS CDK v2 installed globally (`npm install -g aws-cdk`)
- pnpm installed (`npm install -g pnpm`)

## Setup

1. Install dependencies:

```bash
pnpm install
```

2. Configure environment variables:

Create a `.env` file in the root of the project with the following variables:

```
AWS_ACCOUNT_ID=your-aws-account-id
AWS_REGION=us-east-1
ENVIRONMENT=dev
```

## Deployment

To deploy the infrastructure:

```bash
# Bootstrap the CDK (only needed once per account/region)
pnpm run bootstrap

# Synthesize the CloudFormation template
pnpm run synth

# Deploy the stack
pnpm run deploy
```

## Infrastructure Components

### Kinesis Stream

- **Stream Name**: `payloop-usage-events`
- **Mode**: Provisioned
- **Shard Count**: 2 (configurable)
- **Retention Period**: 7 days (168 hours)
- **Encryption**: KMS with customer-managed key
- **Monitoring**: Enhanced monitoring with shard-level metrics

### CloudWatch Alarms

The following CloudWatch alarms are configured:

1. **Shard Utilization**: Alerts when shard utilization exceeds 80%
2. **PutRecord Failures**: Alerts when PutRecord success rate drops below 95%
3. **Iterator Age**: Alerts when iterator age exceeds 1 hour (consumer lag)

### IAM Role

An IAM role is created for ECS tasks to access the Kinesis stream:

- **Role Name**: Generated by CloudFormation
- **Permissions**: Read/Write access to the Kinesis stream and KMS key

## Integration with Payloop

To use the Kinesis stream in Payloop:

1. Set the `EVENT_PUBLISHER` environment variable to `kinesis` in your Payloop environment.
2. Ensure the ECS task role has the permissions defined in this CDK stack.
3. Set the following environment variables in your Payloop environment:
   - `AWS_REGION`: The AWS region where the Kinesis stream is deployed
   - `KINESIS_STREAM_PREFIX`: The prefix for Kinesis stream names (default: `payloop-`)
   - `KINESIS_RETRY_ATTEMPTS`: Number of retry attempts (default: 3)
   - `KINESIS_PROCESSING_TIMEOUT`: Processing timeout in seconds (default: 30)
   - `KINESIS_ENABLE_METRICS`: Enable CloudWatch metrics (default: true)

## Cleanup

To destroy the infrastructure:

```bash
pnpm run destroy
```

## Customization

You can customize the infrastructure by modifying the following files:

- `src/stacks/kinesis-stack.ts`: Kinesis stream configuration
- `src/app.ts`: CDK app configuration
